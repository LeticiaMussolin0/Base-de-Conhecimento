Função que permite emitir Ordens de Carregamento baseado em um Contrato de Frete integrado na Plataforma Transvale.

Após a seleção do Contrato/Sequencial, os dados necessários para emissão da Ordem de Carregamento são:

1. Valor da Mercadoria;
2. Quantidade;
3. Peso;
4. Observações;
5. Placa do Veículo
  • A Placa do Veículo é consultada na Base do Maxicon para determinar os dados do Conjunto. Caso não exista Junção cadastrada será retornado mensagem de alerta;

    • Antes de gerar a Ordem de Carregamento, é consultado e validado os dados cadastrais do Conjunto informado na Gerenciadora de Risco. Caso não exista será retornado mensagem de alerta;

    Será gerado o PDF da Ordem de Carregamento, que poderá ser enviado por e-mail ou WhatsApp.

    Antes de finalizar o Processo, o Sistema pergunta se deseja Gerar Ordem de Carregamento para Frete de Retorno. Se Sim, são listados todos os contratos que possuem como Destino a Origem do Contrato Selecionado. Os contratos que atendam ao critério de busca são listados em tela e o processo se repete para emissão da Ordem de Carregamento de Retorno.

    Nas opções dos botões de Ação existe mecanismo de Previsão de Tempo. Através do Controle de Frete selecionado, é apresentado para a ORIGEM do Embarque, a previsão do tempo, considerando 5 dias: dia corrente e os próximos 4.

    PONTO DE ATENÇÃO: TODAS AS ORDENS DE CARREGAMENTO EMITIDA SERÃO LISTADAS DENTRO DO FLUXO DE EMISSÃO DE CTe (Central de Manifesto Compartilhado). TODAS as ORDENS DE CARREGAMENTOS EMITIDAS SÃO INTEGRADOS A BASE DO MAXICON.

    Acesse em: > Gestão de Contratos > Ordem de Carregamento > Home.

    Função que permite o Pagamento das Cartas Fretes Transvale (Adiantamento e ou Saldo).

    Os dados de entrada são:

    1. Código de Barras CFe/CTe: Pesquisa da Carta Frete ou DACTe através do código de barras via leitor de documentos;
    2. Parcela: 1-Adiantamento ou 2-Saldo;
      Quando Parcela não informada, o sistema buscará a parcela que consta aberta;
    3. CT-e: Número do CT-e;
    4. Série: Série do CT-e;
    5. CPF: CPF do Motorista;

    Quando o Campo Código de Barras é informado os outros campos são dispensáveis;
    Se o Código de Barras não for informado, os campos Parcela, CT-e, Série e CPF são obrigatórios.
    O CTe localizado é listado no GRID (FIGURA 1)

    Tela de Acerto de Pagamento de Carta Frete. Acesse: Gestão de Contratos > Pagamento Carta Frete > Home.


    Para a execução do Pagamento basta clicar na linha do CT-e e informar o Peso Destino conforme documentação apresentada.

    Se Saldo, o Peso Destino deverá ser informado.
  • Informar o Peso Destino conforme Ticket de Balança apresentada;
  • As atuais regras já existentes para o Pagamento de Saldo permanecem inalteradas;
  • Se Peso Destino MAIOR que Peso Carregamento, o valor do Peso deverá ser repetido.
      • A Plataforma não permite Peso Destino Superior que Peso Destino;
  • Após informar o Peso de Destino, o botão Recalcular deverá ser acionado para recalcular o valor do Saldo;
      • Nesse momento todas as regras parametrizadas no Sistema (Contrato, Clifor etc) serão aplicadas;
  • O Valor a Pagar será exibido bem como os descontos (Cadastro Gerenciadora de Risco, por exemplo);
  • Se precisar, os Adicionais de Frete poderão ser usados para compor o Pagamento;
      • Os Adicionais de Frete são os mesmos já usados e com as mesmas regras de aplicação;
  • Informar o Comprovante de Descarga, OBRIGATÓRIO para o processo;
      • Sem o Comprovante anexado, o Pagamento não é realizado;

    O Cadastro de Frete possui as seguintes regras:

    1. Cadastrar Frete: Permite a inserção de um novo Frete;
    2. Exportar para Excel: lista todos os Fretes cadastrados no formato Excel;
    3. Exportar para PDF: lista todos os Fretes cadastrados no formato PDF;
    4. O Cadastro de Frete possui validade de 48h. Passado esse período ele é excluído automaticamente do sistema;
    5. É possível editar o Cadastro de Frete, clicando sobre o registro desejado. O campo passível de edição é o valor do Frete;
    6. Ao editar o Frete, o sistema aplica mais 48h de validade considerando a data e hora da edição;
    É possível também, se desejado, a exclusão do Cadastro de Frete através de botão de ação "Apagar".

    O Cadastro de Frete possui os seguintes campos:

    1. Origem: Cidade de Origem;
    2. Destino: Cidade de Destino;
    3. Distância: Distância em KM entre Origem x Destino;
      O cálculo da distância é realizado através do Google Maps;
    4. Produto: Lista de Produtos (Natureza de Carga), conforme cadastros já existentes no Maxicon;
    5. Espécie: Espécie do Produto, conforme cadastros já existentes no Maxicon;
    6. Frete Motorista R$: Valor do Frete Motorista conforme Contrato;
    7. Valor a Combinar: usado quando o Frete Motorista é negociável;
    8. Empresa Contrato: Filial detentora/responsável pelo Contrato de Frete;
    9. Observação: campo de uso livre para informar dados adicionais e relevantes ao Contrato de Frete;

    O Frete Cadastro é listado no GRID e disponível para consulta no APP.

    Para adicionar, clique no botão: "Cadastrar Frete".

    Para editar o frete cadastrado, dê um duplo click no resultado.

    Função que permite a interação entre Motorista Terceiro x Filial para o atendimento de uma solicitação de Frete.

    A partir da pesquisa realizada pelo Motorista Terceiro via APP, as seguintes ações serão disparadas:

    1. Quero Frete:

  • O APP permite ligação direta para a Filial do Contrato via Dial ou WhatsApp;
  • Nesse momento é realizado o atendimento e negociação;

    • 2. Solicitar Atendimento:
  • Será gerado um registro nessa tela para o retorno via ligação ao Motorista para atendimento e negociação;
      • 1. Nesse momento poderá ser emitido a Ordem de Carregamento ou;
      2. Informar o motivo da recusa do Frete;
    O sistema controla o tempo de vida do registro e o tempo de atendimento;
    O atendimento poderá ser realizado via WhatsApp diretamente na Plataforma.

    PONTO DE ATENÇÃO IMPORTANTE: O CADASTRO DO MOTORISTA DEVE ESTAR CORRETO, POIS O TELEFONE CONSIDERADO SERÁ O INFORMADO DENTRO DO PROCESSO DE CADASTRO DE MOTORISTA

    3. Para o atendimento, basta selecionar as opções disponíveis no Botão de Ação que são:

  • Adicionar Ordem de Carregamento: Habilita o formulário de emissão de Ordem de Carregamento;
      • 1. O Contrato deverá ser informado;
      2. A Variante deverá ser selecionada;
      3. Preencher os campos do formulário e gerar a Ordem de Carregamento;
    Finalizar: finaliza o Atendimento. Usado quando a negociação não é concluída;
      1. Informar o motivo e preencher uma observação (opcional);
      2. Motivo de Recusa mediante Cadastro;
    Contatar via WhatsApp: será aberto o WhatsApp Web chamando o telefone do Motorista;
    Excluir: Excluir o registro de atendimento;
      1. Baseado em Perfil de Acesso.

    Função para a Gestão e envio dos Malotes entre Filiais x Matriz.

    Os documentos que deverão ser considerados neste processo são TODOS os CT-e e Carta Frete que precisam ser enviados à Matriz.

    O Processo está divido da seguinte forma:

    Emissão: Processo de Criação do Malote. O Sistema permite Consultar o Documento através do Scanner ou manualmente informando os dados de acesso (CTe + Série + CPF do Motorista);
    1. O Documento será consultado na base do Maxicon e listado em tela;
    2. Informar qual a parcela (1- Adiantamento, 2 - Saldo ou 3 - Ambos)
    3. Repetir o passo de identificação para a montagem da lista de documentos;
    4. Informar a Filial de Emissão do Malote;
    5. Ao final, o sistema gerará uma Capa com um Código de Barras com os dados de identificação da Filial x Total de Documentos que deverá ser anexado a todos os Documentos e postados no Malote;
    Conferência: Processo de Conferência "Cega" dos documentos Emitidos. Esse processo poderá ser realizado pela Matriz ou Filial;
    1. O sistema validará os documentos lidos e marcará a linha na cor verde os que foram "encontrados";
    2. Os documentos não encontrados na lista serão listados e retornados por e-mail ao usuário Emissor para ciência e correção;
    3. O Malote permanecerá INCONSISTENTE até que os documentos não identificados sejam reenviados e conferidos;
    Transferência: permite a transferência dos Malotes Conferidos entre Filiais ou Matriz;
    1. O sistema permite Unir vários Malotes Conferidos em um ÚNICO adicionando novos documentos, gerando um ÚNICO PROTOCOLO (Capa de Malote);
    2. Informar qual a Filial que deverá ser TRANSFERIDO;
    3. A Filial marcada na TRANSFERÊNCIA efetuará a Conferência seguindo os mesmos procedimentos já citados.

    Para exibir o controle de emissão, acesse: Comprovante de Documento > Controle de Malote > Emissão.

    Funcionalidade de uso exclusivo dos Postos Conveniados que operam o Vale Diesel Transvale. Tem como objetivo validar o Voucher emitido para uso do Vale Diesel.

    Após a emissão do CT-e com o Vale Diesel é impresso um Voucher com os seguintes dados:

  • Dados cadastrais do Motorista + Foto do Motorista;
    • A foto será retornada desde que no cadastro do Motorista exista a foto;
  • Data de Emissão;
  • Valor do Vale Diesel
    • Por processo, o Posto Conveniado, antes do abastecimento, deverá consultar o Voucher validando e conferindo os dados.

    Se o Voucher por ventura já estiver utilizado o sistema retornará com os dados da utilização (Data, Valor e Posto), bloqueando qualquer ação do Posto.
